Вітаю з успішним налаштуванням Git! Це важлива віха у вашій підготовці до кар'єри в IT. Оскільки ви зараз активно вивчаєте Manual Testing та готуєтеся до травневого іспиту, володіння базовою термінологією англійською мовою є критично необхідним для складання тест-кейсів та розуміння документації.

Ось 20 найпоширеніших термінів, які знадобляться вам у роботі та на іспиті:

Робота з дефектами та результатами
Bug — помилка або дефект у програмі.

Issue — загальна назва для проблеми або запиту, які потребують вирішення.

Defect — невідповідність між фактичним результатом та вимогами.

Failure — збій у роботі системи, помітний користувачеві.

Pass / Fail — статус тесту: «пройдено» або «не пройдено».

Severity — серйозність дефекту (наскільки він критичний для роботи системи).

Priority — пріоритетність виправлення (як швидко розробник має це виправити).

Тестова документація
Requirement — вимога, опис того, як програма повинна працювати.

Test Case — опис кроків для перевірки конкретної функції.

Test Suite — набір тест-кейсів, об'єднаних за певною ознакою.

Checklist — список параметрів або функцій, які потрібно перевірити.

Actual Result — те, що ви реально побачили в програмі під час тесту.

Expected Result — те, що мало б статися згідно з документацією.

Процес тестування
Precondition — передумова, стан системи перед початком тесту.

Step — конкретна дія користувача під час виконання тесту.

Environment — середовище (залізо, ОС, браузер), де проводиться тестування.

Build — конкретна версія програми, яку ви тестуєте.

Release — випуск готової версії продукту для користувачів.

End-to-End (E2E) — тестування повного ланцюжка дій користувача від початку до кінця.

Usability — зручність використання програми для кінцевого користувача.
